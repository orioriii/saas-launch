# Project Instructions（Claude Code 向け）

このリポジトリは「SaaS ローンチ・ハーネス」です。非エンジニアがコマンド1つで、自分の SaaS を
Cloudflare（バックエンド）と Vercel（フロント）にデプロイできるようにするツールです。

Claude Code / AI エージェントがユーザーを支援するときは、次を守ってください。

- ユーザーは非エンジニアだと思って、専門用語を避けて言語化してください。
- ゴール（デプロイ完了・連携完了）から外れる提案をしないでください。ゴールに進む提案を必ずしてください。
- 回答には必ず次の2つを含めてください:
  - **今の進捗を全体像から整理するとこれ**（`saas-launch status` の結果を要約）
  - **次のタスクはこれ**（具体的な1コマンド or 1操作）
- デプロイや連携がうまくいかない時は、まず `check-integrations` スキル（`saas-launch doctor`）で
  どこの連携が足りていないかを調べてから、直し方を1つずつ案内してください。
- 破壊的な操作（削除・上書き・本番への再デプロイ）は、実行前に必ずユーザーに確認してください。

## よく使うコマンド

- `saas-launch setup` … デプロイを進める（失敗しても続きから再開できる）
- `saas-launch status` … 今どのステップにいるかを表示
- `saas-launch doctor` … 連携不足を診断（`--json` で機械可読）

## スキル

- `skills/check-integrations/` … システムのどこの連携が足りていないかを調査するスキル。
