# Vercel の登録とログイン

フロントエンド（画面）のデプロイ先です。**Hobby プランは無料**で、個人・検証用途には十分です。

## 1. アカウントを作る

1. ブラウザで <https://vercel.com/signup> を開く
2. サインアップ方法を選ぶ（どれでも OK）
   - **Continue with GitHub** … 最もおすすめ。GitHub 連携でデプロイが楽になります
   - GitLab / Bitbucket / メールアドレス でも可
3. 画面の指示に従って登録を完了します

> すでにアカウントを持っている場合は、この手順は不要です。

## 2. CLI（vercel）でログインする

`saas-launch setup` を実行すると、次を自動で行います（手動でも実行できます）:

```bash
npx vercel login
```

- メールアドレス、または GitHub 等での認証画面が出ます。
- 案内に従って認証すると、ターミナルにログイン完了が表示されます。

## 3. ログインできたか確認する

```bash
npx vercel whoami
```

- あなたのユーザー名（またはチーム名）が表示されれば成功です。

## 4. プロジェクトのリンクについて

ハーネスは、フロントのディレクトリを Vercel の「プロジェクト」に結び付けます（`vercel link`）。
これも `saas-launch setup` が自動で行いますが、手動でやる場合は次のとおりです:

```bash
cd apps/web            # フロントのディレクトリへ
npx vercel link        # 画面の質問に答えてプロジェクトを作成/選択
```

## うまくいかないとき

| 症状 | 対処 |
|------|------|
| `vercel` が見つからない | `npx vercel ...` の形で実行。Node.js が入っているか確認 |
| ログインメールが届かない | 迷惑メールを確認。数分待って再送 |
| デプロイ時に env が無いと言われる | `saas-launch setup` の「フロント環境変数の設定」を先に済ませる |
| ビルドが失敗する | フロントの `package.json` の `build` スクリプトがローカルで通るか確認 |

ログインができたら、`saas-launch setup` に戻って続きから進めてください。
