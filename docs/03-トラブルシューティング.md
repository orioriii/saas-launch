# トラブルシューティング

## まず最初に

うまくいかないときは、次の2つを覚えておいてください。

1. **失敗しても、もう一度同じコマンドを実行すれば続きから再開できます。**
   ```bash
   saas-launch setup
   ```
   成功済みのステップは自動でスキップされ、失敗した所からやり直します。

2. **どこの連携が足りていないかは、診断コマンドで分かります。**
   ```bash
   saas-launch doctor
   ```
   ✅（OK）/ ⚠️（要確認）/ ❌（未連携）と、それぞれの「直し方」が表示されます。

---

## よくあるエラーと対処

### 「Cloudflare ログイン未完了」で止まる

- `npx wrangler login` を実行し、ブラウザで **Allow** を押してください。
- その後 `npx wrangler whoami` で確認 → `saas-launch setup` を再実行。

### バックエンドのデプロイが失敗する

- `wrangler.toml` の `name` / `account_id` / (D1使用時は)`database_id` が正しいか確認。
- D1 を使う設定なら、先に「D1 データベース作成」ステップが完了しているか確認。
- エラーメッセージを直したら `saas-launch setup` を再実行（続きから進みます）。

### シークレット登録（wrangler secret put）が失敗する

- Worker がまだ一度もデプロイされていないと失敗することがあります。
- その場合は、いったんバックエンドをデプロイしてから、`saas-launch setup` を再実行してください。

### 「フロント→バックエンド 連携設定」が ❌（doctor）

- フロント画面がバックエンドの場所（URL）を知らない状態です。
- 直し方（doctor の fix にも表示されます）:
  ```bash
  cd apps/web
  echo "https://xxx.workers.dev" | npx vercel env add NEXT_PUBLIC_API_URL production
  npx vercel deploy --prod --yes
  ```
- もしくは `saas-launch setup` を再実行すれば自動で埋められます。

### 「バック⇄フロント CORS 連携」が ❌（doctor）

- ブラウザからの API 呼び出しがブロックされる状態です（CORS）。
- 直し方:
  ```bash
  cd apps/api
  echo "https://xxx.vercel.app" | npx wrangler secret put ALLOWED_ORIGIN
  npx wrangler deploy
  ```
- `saas-launch setup` の最後の「連携」ステップでも自動的に行われます。
- なお、CORS の実際の許可はバックエンド側のコードが `ALLOWED_ORIGIN` を読んで
  `Access-Control-Allow-Origin` を返す実装になっている必要があります。

### Vercel のデプロイが失敗する

- 環境変数不足 → 「フロント環境変数の設定」ステップを先に済ませる。
- ビルドエラー → フロントの `package.json` の `build` がローカルで通るか確認。
- プロジェクト未リンク → `cd apps/web && npx vercel link`。

## 状態をリセットしたいとき

最初からやり直したい場合は、状態ファイルを削除します（クラウド上のリソースは消えません）。

```bash
rm .saas-launch-state.json
saas-launch setup
```

> クラウド上に作った Worker / D1 / Vercel プロジェクトそのものを消したい場合は、
> それぞれのダッシュボード、または `wrangler` / `vercel` のコマンドで削除してください。

---

## それでも解決しないとき

`saas-launch doctor --json` の出力を添えて、AI エージェント（Claude Code）に相談してください。
`check-integrations` スキルが、どこが繋がっていないかを整理して次の一手を提案します。
