# はじめに — 全体像と設定ファイル

## このツールは何をするのか

あなたの SaaS（Web サービス）を、次の2つのクラウドに公開（デプロイ）します。

- **バックエンド**（データを処理する側）→ Cloudflare Workers
- **フロントエンド**（画面）→ Vercel

そして、両者が正しく**連携**する（画面からバックエンドを呼べる）ところまで面倒を見ます。

エンジニアでなくても、AI エージェントの質問に答えていくだけで進められるように作られています。

## 全体の流れ（8ステップ）

```
Cloudflare（バックエンド）
  1. Cloudflare 登録 & ログイン
  2. D1 データベース作成          ← 使う設定のときだけ
  3. シークレット設定（APIキー等）
  4. バックエンドをデプロイ        → https://xxx.workers.dev

Vercel（フロントエンド）
  5. Vercel 登録 & ログイン
  6. フロント環境変数の設定        ← バックエンドURLを自動で渡す
  7. フロントをデプロイ            → https://xxx.vercel.app

連携
  8. バック⇄フロント連携（CORS）   ← フロントURLをバックエンドに許可させる
```

「今どこにいるか」はいつでも `saas-launch status` で確認できます。

## 設定ファイル `harness.config.json`

「何を・どこに」デプロイするかを書くファイルです。1つ用意すれば、あとはハーネスが
この内容に合わせてステップを組み立てます。

### backend（バックエンド）

| 項目 | 説明 |
|------|------|
| `dir` | wrangler を実行するディレクトリ（例 `apps/api`） |
| `deployCommand` | デプロイに使うコマンド（既定 `npx wrangler deploy`） |
| `healthPath` | 生存確認に使うパス（例 `/health`）。診断（doctor）で使う |
| `d1.enabled` | Cloudflare D1（データベース）を使うか |
| `d1.databaseName` | 作成する D1 の名前 |
| `d1.wranglerBinding` | wrangler.toml 上の binding 名（既定 `DB`） |
| `secrets[]` | 登録するシークレット（APIキーなど）の一覧 |

#### secrets の各項目

| 項目 | 説明 |
|------|------|
| `name` | シークレット名（例 `API_KEY`） |
| `prompt` | ヒアリング時の質問文 |
| `howto` | どこで取得するかの案内（例: Stripe ダッシュボード → …） |
| `generate` | `true` なら空入力時に安全なランダム値を自動生成 |
| `optional` | `true` なら未入力でもデプロイを続行 |

### frontend（フロントエンド）

| 項目 | 説明 |
|------|------|
| `dir` | vercel を実行するディレクトリ（例 `apps/web`） |
| `framework` | 使っているフレームワーク（任意、例 `nextjs`） |
| `env[]` | Vercel に登録する環境変数の一覧 |

#### env の各項目

| 項目 | 説明 |
|------|------|
| `name` | 環境変数名（例 `NEXT_PUBLIC_API_URL`） |
| `fromBackendUrl` | `true` なら**バックエンドのデプロイ URL を自動で値にする**（連携の要） |
| `prompt` / `howto` | 手入力が必要な場合のヒアリング文言 |

### wiring（連携）

| 項目 | 説明 |
|------|------|
| `backendAllowedOriginVar` | バックエンド側で「許可するフロントのオリジン」を保持する変数名（既定 `ALLOWED_ORIGIN`）。ここに Vercel の本番 URL を登録して CORS を疎通させる |

## 状態ファイル `.saas-launch-state.json`

進捗と、入力済みの設定値（URL など）が保存されます。**手で編集する必要は普通ありません。**

- 途中で失敗しても、このファイルのおかげで「続きから」再開できます。
- APIキー等のシークレットの**値は保存されません**（登録が完了した「名前」だけを記録します）。
  シークレット登録の途中で中断した場合、未登録の項目だけ再入力になります。
- 個別環境の情報を含むため、Git にはコミットされません（`.gitignore` 済み）。
- すべて成功すると自動で削除されます。
