# 連携不足の調査スキル（check-integrations）

このスキルは、`saas-launch doctor` を実行して**システムのどこの連携が足りていないか**を突き止め、
非エンジニアのユーザーにも分かる言葉で説明し、次にやるべきことを1つずつ案内するためのものです。

## 前提

- 対象プロジェクトのルート（`harness.config.json` がある場所）で作業していること。
- ハーネス本体がビルド済みであること（未ビルドなら `npm run build`、または `npx tsx` 経由で実行）。

## 手順

1. **診断を機械可読な形で取得する**

   まず JSON で診断結果を取得します（値を解釈しやすいため）。

   ```bash
   saas-launch doctor --json
   ```

   ハーネスを開発中でグローバルに入っていない場合は、ハーネスのディレクトリから:

   ```bash
   npx tsx src/index.ts doctor --json    # もしくは: node dist/index.js doctor --json
   ```

2. **JSON を解釈する**

   返ってくる `DoctorReport` は次の形です:

   ```jsonc
   {
     "projectName": "my-saas",
     "backendUrl": "https://xxx.workers.dev",
     "frontendUrl": "https://xxx.vercel.app",
     "summary": { "ok": 4, "warn": 1, "fail": 2, "skip": 0 },
     "checks": [
       { "id": "cloudflare-login", "label": "Cloudflare ログイン", "status": "ok", "detail": "..." },
       { "id": "frontend-env", "label": "フロント→バックエンド 連携設定", "status": "fail",
         "detail": "フロントに未設定の環境変数: NEXT_PUBLIC_API_URL（＝フロントがバックエンドの場所を知らない状態）",
         "fix": "..." }
     ]
   }
   ```

   各 `status` の意味:
   - `ok` … 連携できている
   - `warn` … 動くかもしれないが要確認
   - `fail` … 連携が足りていない（要対応）
   - `skip` … その項目は設定上チェック不要

3. **ユーザーに分かりやすく報告する**

   - まず**全体像**を1〜2行で（例: 「7項目中、5つはOK、2つが未連携です」）。
   - 次に `fail` と `warn` の項目だけを、**専門用語を避けて**説明する。
     - 例（frontend-env が fail のとき）:
       「フロント画面が“バックエンドの住所”をまだ知りません。この設定をすると画面からデータを取れるようになります。」
   - 各項目の `fix` を、**そのまま実行できるコマンド**として1つずつ提示する。
   - 最後に「まとめて直すなら `saas-launch setup` を実行すれば、続きから自動で埋められます」と伝える。

4. **回答フォーマット（AGENTS.md の方針に従う）**

   回答には必ず次を含めること:
   - **今の進捗を全体像から整理するとこれ**: OK / 未連携の数と、何が繋がっていて何が繋がっていないか。
   - **次のタスクはこれ**: `fail` のうち最優先の1つと、その具体的コマンド。

## チェックしている連携（参考）

| id | 何を見ているか | 足りないと何が起きるか |
|----|----------------|------------------------|
| `cloudflare-login` | wrangler にログイン済みか | バックエンドをデプロイできない |
| `backend-reachable` | Worker が公開され応答するか | API が存在しない |
| `backend-secrets` | 必要な APIキー等が登録済みか | 認証や外部連携が動かない |
| `vercel-login` | vercel にログイン済みか | フロントをデプロイできない |
| `frontend-reachable` | フロントが公開され応答するか | 画面が表示されない |
| `frontend-env` | フロントにバックエンド URL が設定済みか | 画面がバックエンドの場所を知らない |
| `cors-wiring` | Worker がフロントのオリジンを許可しているか | ブラウザから API 呼び出しがブロックされる |

## 注意

- このスキルは**調査（読み取り）専用**。設定を勝手に変更しない。
- 修正は必ずユーザーに提案し、ユーザーの了承のもとで `saas-launch setup` か個別コマンドを実行する。
